Transformative Applications Across Roles

Architecture Review

"As a senior architect for a FinTech company, provide a detailed review of this new microservice. **Requirements**: Assess its scalability using AWS Lambda, identify potential security risks related to data encryption, and evaluate its maintainability based on our team's Python style guide. **Format**: A bulleted list of findings, with a severity rating for each issue."

Code Quality Analysis

"Act as a code quality specialist. Analyze the provided JavaScript code for our new e-commerce checkout module. **Requirements**: Identify common technical debt patterns (e.g., God objects, duplicated logic, complex conditionals). **Audience**: Junior developers on the team. **Format**: Provide a list of the top 5 issues, with a clear explanation of why each is an issue and a refactoring suggestion for each one. Include a short summary at the end."

Incident Response

"Act as a DevOps engineer. Analyze the attached logs from our production Kubernetes cluster following a service degradation. **Requirements**: Identify the potential root cause of the spike in latency, pinpoint the specific service or container that failed, and suggest a 3-step investigation plan for the on-call team. **Format**: A numbered list for the investigation steps, a short paragraph for the root cause summary, and highlight the service name."

Test Strategy Generation

"As a test lead, create a risk-based testing strategy for a new B2B payment processing microservice. **Requirements**: Identify the highest risk areas (e.g., transaction integrity, data security, third-party API calls) and provide specific test cases for each risk level (high, medium, low). **Audience**: The development team and product manager. **Format**: A table with columns for 'Risk Area', 'Description', and 'Example Test Cases'."

Edge Case Discovery

"Act as a security tester. Analyze the provided OpenAPI spec for our user registration API. **Requirements**: Identify potential security vulnerabilities, including input validation bypass, injection flaws, and weak authentication schemes. **Format**: A markdown list with headings for each vulnerability type, a brief description of the risk, and a corresponding Gherkin scenario to test it."

Automated Documentation

"Act as an automated documentation tool. Given a test run summary, generate a concise report that explains the business impact of the failed tests. **Requirements**: Focus on how the failures affect key user flows (e.g., 'user cannot add item to cart'). **Audience**: Product owners and business stakeholders. **Format': A two-paragraph summary, followed by a bulleted list of failed user stories."

Pattern Recognition

"Act as a data analyst. Analyze the last 5 sprint retrospectives from the 'Project X' team. **Requirements**: Identify recurring themes or systemic issues that have been mentioned in at least three of the retrospectives. Categorize these issues into 'Process', 'Communication', and 'Technical Debt'. **Format': A short, high-level summary paragraph followed by a detailed list of findings for each category."

Stakeholder Communication

"Act as a technical project manager. Translate the following technical blockers for the 'User Profile' feature into a clear, non-technical email update for executive stakeholders. **Requirements**: Focus on the business impact, estimated timeline delays, and a proposed mitigation plan. **Audience**: Non-technical executives. **Format': An email draft with a clear subject line, a concise summary, and a bulleted list of key takeaways."

Team Development

"Act as a team lead. Given a list of new technologies for the upcoming quarter (e.g., React Server Components, Rust for backend, GraphQL), design a professional development plan for my team of 4 front-end engineers. **Requirements**: The plan should balance individual learning with project needs and include an estimated time commitment for each item. **Format': A table with columns for 'Technology', 'Skill Level', 'Learning Resource', and 'Time Estimate'."

Trust but Verify: The Professional Standard

AI is a powerful tool, but its output requires our critical judgment. It's our responsibility to validate, audit, and own the results. Here are the core principles of professional AI usage.

Logic Check

Does this make sense? Does it solve the right problem? Question AI's suggestions just as you would a junior dev's code.

Context Verification

Does this solution fit our specific technical stack, team conventions, and business constraints?

Standard Compliance

Is the output secure? Is it maintainable? Does it follow our quality and ethical guidelines?

Risk Assessment

What are the potential negative side effects? An AI-generated optimization might introduce a new bug or security vulnerability.
